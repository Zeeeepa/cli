# Getting Started

TestDriver is a powerful tool for automating tests. This guide will walk you through the process of setting up and running your first test.

## Prerequisites

Before you begin, ensure you have the following:

- A working installation of TestDriver.
- Basic knowledge of the command line.
- A test environment set up.

## Installation

To install TestDriver, follow these steps:

1. Download the latest version from the official website.
2. Run the installer and follow the on-screen instructions.
3. Verify the installation by running `testdriver --version` in your terminal.

## Writing Your First Test

Once TestDriver is installed, you can write your first test. Here's a simple example:

```plaintext
testdriver run --test example_test
```

This command runs the `example_test` file. Make sure the file exists in your test directory.

## Running Tests

To run tests, use the following command:

```plaintext
testdriver run --all
```

This will execute all tests in your test directory.

## Troubleshooting

If you encounter issues, check the following:

- Ensure TestDriver is installed correctly.
- Verify your test files are in the correct directory.
- Consult the TestDriver documentation for more help.

## Next Steps

Now that you've run your first test, explore advanced features of TestDriver to enhance your testing workflow.
